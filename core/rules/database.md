# CORE-DB：数据库规则

<!-- rule-id: CORE-DB-001 -->
- 数据库结构只能通过 migration 演进，禁止服务启动时自动改表。

<!-- rule-id: CORE-DB-002 -->
- 已发布 migration 不允许回改，只能追加新版本。

<!-- rule-id: CORE-DB-003 -->
- 代码依赖的新字段必须与 migration 同批交付，并能在干净数据库上完成初始化。

<!-- rule-id: CORE-DB-004 -->
- 状态字段保存稳定 code，不保存展示文案。

<!-- rule-id: CORE-DB-005 -->
- 新增列表、关联或查询路径时评估索引；修改字段语义前确认历史数据、兼容窗口和回滚方案。

<!-- rule-id: CORE-DB-006 -->
- 写操作必须明确事务边界，跨外部系统的一致性策略必须显式记录。
